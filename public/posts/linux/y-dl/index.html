Linux学习笔记

January 7, 2026 · XY

为什么选择 Linux 跑深度学习？#

在Windows下配置深度学习环境经常遇到路径冲突、版本冲突和驱动问题。通过 WSL2可以在保留Windows日常办公方便性的同时，拥有Linux原生的开发体验，以便过度到Linux系统，选择理由如下：

一、 四种主流安装方案#

安装方式 所需软件/硬件条件 优点 缺点 适用场景
1. WSL2 (Windows子系统) Win10/11, 开启虚拟化, 建议16G内存 无需重启切换系统，文件互通，性能损耗极小 GUI（图形界面）配置较繁琐，部分底层硬件驱动受限 兼顾办公与深度学习开发
2. 双系统 (Dual Boot) 独立磁盘分区 (50G+), U盘启动盘 完全释放硬件性能，独立环境 切换系统需重启，安装风险较高（易引导损坏） 适合长时间训练大型模型
3. 虚拟机 (VMware/VBox) 强劲CPU, 32G+内存, 开启虚拟化 安全、随便折腾、可快照备份 无法直接调用NVIDIA显卡，性能损耗大 仅限学习Linux基础命令，不适合深度学习
4. Docker容器 已装好的Linux或WSL2 环境隔离，防止配坏驱动，方便部署 有一定学习成本 团队协作、多版本框架切换

二、 方案优缺点分析#

1. WSL2 (Windows Subsystem for Linux 2) —— 推荐新手/混合开发#

微软近年来力推的方案，在Windows内运行一个Linux内核。

  • 硬件要求：普通的现代Windows电脑，需在BIOS里开启 Virtualization（虚拟化）。
  • 软件要求：Windows 10 版本 19041 或更高，Windows 11。
  • 优势：WSL2已完美支持NVIDIA CUDA。可在Windows里装好显卡驱动，WSL2里的Ubuntu能直接调用GPU进行训练。

2. 双系统 (Windows + Ubuntu) —— 推荐专业开发者#

硬盘上划一块给Linux，开机时选择进入哪个系统。

  • 硬件要求：至少预留一个50GB-200GB的磁盘分区（推荐SSD硬盘）。
  • 软件要求：使用 Rufus 或 Ventoy 制作Ubuntu ISO启动U盘。
  • 优点：没有中间商赚差价，显卡驱动最稳定，内存利用率最高。
  • 建议型号：主流用 Ubuntu 20.04 LTS 或 22.04 LTS。

3. 虚拟机 (VMware/VirtualBox) —— 不推荐#

虚拟机在Windows上再运行一个操作系统，性能损耗大，显卡驱动配置复杂。

4. Docker容器 —— 团队推荐#

Docker容器在Linux上再运行一个操作系统，适合团队协作，不适合个人使用。

三、 深度学习/计算机视觉领域的建议#

如果你是跑深度学习代码（如 PyTorch, TensorFlow, OpenCV）：

  1. 第一选择：WSL2 + Ubuntu 22.04

    • 理由：可以在Windows看网页教程、用微信交流，另一边在Linux终端里跑模型。
    • 配置流程：安装WSL2 -> 安装Ubuntu -> 在Windows安装最新NVIDIA驱动 -> 在Ubuntu里安装 Miniconda -> 安装 PyTorch

  2. 第二选择：原生双系统 (Ubuntu 22.04)

    • 理由：需要连续几天几夜挂机跑大模型，WSL2 会有内存泄露或稳定性问题，此时原生系统更靠谱。

  3. 环境管理工具 无论用哪种方法，安装完Linux后，不要直接在系统环境装库。

    • 建议安装：Miniconda，结合pixi使用。
    • 原因：让你为每个项目创建独立的虚拟环境。
